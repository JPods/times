# Route-Time — Retrospection 2026-04-24

## What We Built

Extended the Route-Time network editor with structure move, region select, and visual
feedback. Server unchanged except for two `api.py` additions.

---

## Problems Encountered and How They Were Solved

### 1. Alt+drag: model moved but map also panned (no net position change)

**Symptom:** Holding Option and dragging a CP marker caused both the map and the
structure to translate together — net result: structure appeared to stay put.

**Root cause:** Leaflet's drag handler listens for `mousedown` on the map container.
`map.dragging.disable()` was called inside a Leaflet `marker.on("mousedown")` handler,
which fires *after* Leaflet's drag handler has already captured the event and attached
its internal `_onMove` / `_onUp` listeners. Disabling dragging at that point was too late.

**Fix:** Added a `document.addEventListener("mousedown", ..., { capture: true })`
handler. The capture phase fires *before* any bubble-phase handlers at the map
container, so Leaflet's drag handler never sees the Alt+mousedown. `stopPropagation()`
in the capture handler is sufficient — no `map.dragging.disable()` needed.

Hover tracking (`_hoverStructSid`) via `marker.on("mouseover/mouseout")` on CP
markers tells the capture handler which structure to grab.

**Files:** `gui/static/app.js`

---

### 2. Alt+drag moved CP squares but not internal lines

**Symptom:** CP badge markers (the coloured squares) translated correctly, but the
polylines making up the internal geometry of the station or circle stayed in place.

**Root cause:** The live-preview update loop only iterated `_cpPropsMap` (CP markers).
Lines had no structure association on the JS side.

**Fix (two parts):**
- `api.py`: added `structure_id` field to line GeoJSON properties for all lines whose
  `line_id` appears in `struct.line_ids`.
- `app.js`: added `_lineStructMap = {}` (line_id → structure_id), populated in
  `_addLineFeature`. Cleared in `_render`. During Alt+drag mousemove, both CP markers
  and all lines in `_lineStructMap` for the dragged structure are translated using
  saved `origLines` snapshots.

**Files:** `gui/api.py`, `gui/static/app.js`

---

### 3. Waypoint drag tool only moved one line of a traffic circle

**Symptom:** In Waypoint mode, clicking and dragging a line inside a traffic circle
only moved that single guideway (or its pair), leaving the rest of the circle behind.

**Root cause:** The waypoint mousedown handler found the nearest line and operated on
`(line, partner)` only — it had no concept of structure membership.

**Fix:** In the waypoint `map.on("mousedown")` handler, after finding the nearest line,
check `_lineStructMap[lid]`. If the line belongs to a structure, redirect to a full
structure move (`_moveState`) instead of the waypoint path. Added guard
`if (_moveState.active) return` to the waypoint mousemove preview so the two paths
don't interfere.

**Files:** `gui/static/app.js`

---

### 4. Shift-drag selection: no visual feedback, failed when starting on a marker

**Symptom (a):** The yellow rubber-band rectangle appeared, but selected items showed
no visual change after release — user could not tell what was captured.

**Symptom (b):** Starting the drag on a marker (e.g., lower-right corner of selection
over a CP) produced no rectangle and no selection, because marker click handlers call
`stopPropagation`, preventing `map.on("mousedown")` from firing.

**Fix (a — visual highlight):**
- After `_selectInBounds`, set CP markers of selected structures to `cp-hit-selected`
  (orange glow) via `mk.setIcon(_cpIcon(props, true))`.
- Set internal lines of selected structures to orange/thick style; save original
  styles in `_sel.origLineStyles` first.
- Dim free-node markers with `mk.setOpacity(0.4)`.
- `_clearSelection()` restores all saved styles from `_sel.origLineStyles`.

**Fix (b — drag from any starting point):**
- Replaced `map.on("mousedown")` for selection with
  `map.getContainer().addEventListener("mousedown", ..., { capture: true })`.
  Capture phase intercepts before marker handlers, so shift-drag works regardless of
  where the drag starts.

**Files:** `gui/static/app.js`

---

### 5. Proximity guard — structures placed too close together

**Added proactively** (not a bug report, but an obvious correctness issue):
- `_check_overlap(new_lat, new_lon, type, exclude_sid)` helper in `api.py` uses
  `vincenty_m` to compare proposed center against all existing structure centers.
- Minimum separation = `footprint(new) + footprint(existing)`:
  - Station footprint: 45 m (half-length 35 m + stub extension 10 m)
  - Circle footprint: 22.5 m (ring radius 7.5 m + stub length 15 m)
- Check is applied in: `POST /network/station`, `POST /network/circle`,
  `POST /network/structure/<sid>/move`.

**Files:** `gui/api.py`

## Key Bindings Reference (current)

| Gesture | Action |
|---------|--------|
| Click map (tool active) | Place station / circle / switch |
| Click CP | Select for connect |
| Click 2nd CP | Connect the pair |
| Ctrl+CP (connected) | Disconnect |
| Alt+drag CP | Move whole structure |
| Waypoint tool + drag line | Move whole structure (if line is internal) |
| Waypoint tool + click guideway | Add routing bend to that guideway pair |
| Right-click guideway | Add aligned bend to both guideways |
| Shift+waypoint handle | Remove waypoint |
| Shift+drag (empty or over marker) | Rubber-band region select |
| Delete / Backspace | Remove selected region |
| Shift+node | Remove single node |
| Ctrl+guideway | Remove guideway pair |
| Esc | Cancel current operation |
